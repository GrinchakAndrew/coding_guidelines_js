Methods and Function Parameters

  • Optional function arguments start with opt_.
    • function f(size1, opt_size2) {

Alias Long Type Names To Improve Readability

  • It is recommended you should use local aliases while addressing an object's property by its full path-name. It is best to match the last part of the type for the local variable names:
    • some.long.namespace.MyClass = function() {
    };
    myapp.main = function() {
        var MyClass = some.long.namespace.MyClass;
        var staticHelper = MyClass.staticHelper;
        staticHelper(new MyClass());
    };
  • You should not use alias for namespaces:
    • myapp.main = function() {
    var MyClass = some.long.namespace.MyClass;	// Good
    };
    myapp.main = function() {
    var namespace = some.long.namespace;		// Bad
    };

Naming Functions as Constructors

  • The first letter in a function name must be uppercased if the function is a constructor. This helps to distinguish constructors (classes) from regular functions. The technique is described in the base style guide.
    • function ClassName() { ... } //Good

Code Formatting

Code Length Limitations

  • Please observe 80 symbols line limitations, when writing your code.

Boolean Values Fromatting

  • A Boolean variable declaration and definition with condition evaluation:
    • var isConditionSatisfied =
        ((isGood || isNice) &&
         !isBad);
    var hasA =
        ((isA || isB) &&
         !isC);

Brackets

  • Because of implicit semicolon insertion, always start your curly brackets on the same line as whatever they're opening and the closing curly bracket – on the next line, respectively.

  • No spaces inside the brackets of the if-statement is nice-to-have.

  • If you decide to employ else or else if conditional statements, please note that else goes on the same line as the closing curly brace of the opening if-statement.
    • if (something) {
        ...
    } else {
        ...
    }
  • Example of a simpler condition placed on one line: brackets enclosure of conditions makes code more readable.
    • // Start boolean variables' names with is or has:
     var hasChildren = (element.childNodes.length > 0);

'For' Loops

  • Please note single spaces in-between reserved words, parenthesis and operators when using for loops. Other loops follow the same rule:
    • for (i = 0; i < collectionLength; i++) {
        ...
    }

Strings

  • Please split long strings into separate lines with a dangling plus (concatenation) sign:
    • var myFunkyHtml =   '"Slartibartfast" , ' +
        '"Zaphod Beeblebrox" , ' +
        '"Ford Prefect" , ' +
        '"Arthur Dent" , ' +
        '"Marvin the Paranoid Android" , ' +
        'the.mice@magrathea.com';

Optional Arguments Use, Optional Variables Redefinition Safeguards

  • Please note the bad example of optional variable usage with commented explanation within the snippet:
    • function operateWithWindow(opt_window) {
      if (arguments.length < 1) {
        opt_window = window;
      }
      // Operating with window through opt_window is confusing because use of the
      // optional argument is expected.
      ...
    }
  • Please consider good example for proper optional variable usage with verification of the number of passed arguments:
    • function operateWithWindow(opt_targetWindow) {
      var targetWindow = (arguments.length < 1) ? window : opt_targetWindow;
      ...
    }

Array And Object Initializers

  • Single-line array and object initializers are allowed when they fit on a line:
    • var arr = [1, 2, 3];			// No space after [ or before ]
    var obj = {a: 1, b: 2, c: 3};		// No space after { or before }
  • Multi-line array initializers and object initializers are indented 2 spaces, just like blocks:
    • // Object initializer
    var inset = {
      top: 10,
      right: 20
      bottom: 15,
      left: 12
    };
    // Array initializer.
    this.rows = [
      '"Slartibartfast" ',
      '"Zaphod Beeblebrox" ',
      '"Ford Prefect" ',
      '"Arthur Dent" ',
      '"Marvin the Paranoid Android" ',
      'the.mice@magrathea.com'
    ];

Function Declaration

  • It is recommended to omit a white space between the reserved word 'function' (or its name) and its parenthesis
    • var f = function() {
        var something = 100;  
        return something; 
    }
    function f() {
        var something = 100;
        return something;
    }

Function Arguments

  • Lengthy function arguments should be listed individually on separate lines
    • // Four-space, one argument per line.  Works with long function names,
    // survives renaming, and emphasizes each argument.
    var doThingThatIsVeryDifficultToExplain = function(
        veryDescriptiveArgumentNumberOne,
        veryDescriptiveArgumentTwo,
        tableModelEventHandlerProxy,
        artichokeDescriptorAdapterIterator) {
        ...
    };
  • If a function is a nested argument for the called function, indentation is equal to 2 spaces:
    • setTimeout(function() {
      console.log('Function called');
    }, 0);
    var names = prefix.something.myExcellentMapFunction(
        verboselyNamedCollectionOfItems,
        function(item) {
          return item.name;
        });
    var styleElement = createStyleElement([
      createCssRule('#content', {
        visibility: 'hidden'
      }),
      createCssRule('#sidebar', {
        display: 'none'
      })
    ].join('\n'));

More on Indentation

  • All wrapped lines should be indented four spaces, except for arrays, object initializers and nested (anonymous) functions.
    • thisIsAVeryLongVariableName =
        hereIsAnEvenLongerOtherFunctionNameThatWillNotFitOnPrevLine();
    thisIsAVeryLongVariableName = 'expressionPartOne' + someMethodThatIsLong() +
        thisIsAnEvenLongerOtherFunctionNameThatCannotBeIndentedMore();
    someValue = this.foo(
        shortArg,
        'Some really long string arg - this is a pretty common case, actually.',
        shorty2,
        this.bar());

Commenting

  • Commenting arguments is optional; however, they do improve readability, because in most cases function signature is not near the function call.
  • Please try not to break your code blocks by excessive commenting.
    • requestContentGeneratorAsync('landing-page', function() {	// Success callback.
      console.log('Request successfully complete.');
    }, 3000, function() {						// Timeout, Time-out callback.
      console.log('No response within 3 seconds.');
    });
    callFunctionWhenConditionSatisfied(function() {		// When content arrives.
      console.log('Content arrived');
    }, function() {						// Has content arrived?
      return (document.getElementById('content') !== null);
    }, 100);					        // Check interval.

Explicit Calls

  • Please, try to write explicit and clear code:
    • // Implicit behavior is not recommended:
    var string = ['foo', 'bar', 'baz'].join();  //Implicitly called comma can confuse other dev(s)
    var string = ['foo', 'bar', 'baz'].join(',');
    doThis() || doThat();				// Bad

Access Operator Code Formatting

  • Example of object's property access operator formatting: placing the dot on a previous line and the accessed property - on the next for lengthy methods:
    • var element = jQuery('#very-very-very-very-very-long-identifier').
        prop({
          target: '_blank',
          title: 'Will be opened in a new tab/window.'
        }).
        css({
          display: 'inline-block',
          'margin-top': '10px',
          'padding-bottom': 0
        }).
        text('Buy now');
  • Example for logically grouping function calls that are working with one and the same element; function calls for the other elements are placed on a separate line:
    • jQuery('#foo').addClass('bar').        // Get #foo and add .bar class to it.
        parent().hide().                   // Get parent of #foo, and hide it.
        next('.baz').show().               // Get .baz next to the parent, and show it.
        prev('.qux').appendTo('#footer');  // And so on...

Using Arrays to Form Complex Arguments:

  • You can use arrays to build complex structures like the one shown on the example below:
    • jQuery([
      '#very-very-very-very-long-identifier',
      '#another-very-very-very-very-long-identifier',
      '#and-one-more-very-very-very-very-long-identifier'
    ].join(',')).each(function() {
      // ...
    });

Condition Evaluation in Return Statements

  • Good examples of conditional evaluation as a 'return' statement:
    • function doSomething() {
      return ((isGood || isNice) &&
              !isBad);
    }
    function doSomething() {
      var isSuccessful =
          ((isGood || isNice) &&
           !isBad);
      return isSuccessful;
    }
  • Bad example of conditional evaluation as a 'return' statement:
    • function doSomething() { 
      return // Ignores lines after the statement, returns undefined.
          ((isGoog || isNice) &&
           !isBad);
    }
